Interface

[ Screenshot: Halaman Purchase Order — tampilkan Draft PO yang dibuat otomatis ]

Kode — frontend/app/api/pos/route.ts

if (newStok <= product.rop) {
  const noPO = `PO-${new Date().getFullYear()}-${String(nextNum).padStart(3, '0')}`
  await prisma.purchaseOrder.create({
    data: {
      noPO,
      supplier:    'Perlu Ditentukan',
      tanggal:     new Date(),
      status:      'Draft',
      createdById: session.id,
      items: { create: [{ productName: product.name, sku: product.sku,
                          qty: product.rop * 2, hargaSatuan: product.hargaBeli }] },
    },
  })
  await prisma.notification.create({
    data: {
      targetRole: 'manager',
      title:      `Auto Draft PO: ${product.name}`,
      message:    `Stok mencapai ROP (sisa ${newStok}). Draft PO ${noPO} dibuat otomatis.`,
      type:       'warning',
      link:       '/dashboard/purchase-order',
    },
  })
}

Interface

Kode — frontend/app/api/stock-opname/[id]/route.ts

// Hitung total variance value
const pct = Math.abs(totalVarianceValue) / totalInventoryValue * 100

// NFR-002: Jika variance > 10%, wajib verifikasi 2FA Manager
if (pct > 10) {
  if (!otp) return NextResponse.json({ requireOTP: true }, { status: 202 })
  const manager = await prisma.user.findFirst({ where: { role: 'manager' } })
  if (!manager?.totpSecret)
    return NextResponse.json({ message: 'Manager belum setup 2FA' }, { status: 400 })
  const ok = verifyTOTP(otp, manager.totpSecret)
  if (!ok) return NextResponse.json({ message: 'Kode OTP tidak valid' }, { status: 401 })
}

Implementasi TOTP — frontend/lib/totp.ts

// Pure Node.js crypto — tanpa library eksternal
import { createHmac, randomBytes } from 'crypto'

export function verifyTOTP(token: string, secret: string): boolean {
  const step = Math.floor(Date.now() / 30_000)
  return [-1, 0, 1].some((w) => getCode(secret, step + w) === token)
}

Fitur LLM — Generate Email Pengadaan ke Supplier (GPT-4o-mini)

Fitur AI generatif yang membantu Manager/Admin membuat email pengadaan formal ke supplier secara otomatis berdasarkan data Purchase Order. Menggunakan model GPT-4o-mini dari OpenAI.

Alur Penggunaan

  1. Manager/Admin buka halaman detail Purchase Order
  2. Klik tombol "Generate Email AI" (icon bintang)
  3. AI membaca data PO (nomor PO, supplier, daftar barang, total nilai)
  4. GPT-4o-mini generate email formal dalam Bahasa Indonesia
  5. Modal tampilkan subject + isi email, dengan tombol Salin & "Buka di Gmail"

Interface

[ Screenshot: Halaman detail PO — tombol "Generate Email AI" ]
[ Screenshot: Modal hasil email — subject, isi email, tombol Salin & Buka di Gmail ]

Kode — frontend/app/api/generate-email/route.ts

const openai = new OpenAI({ apiKey: process.env.OPENAI_API_KEY })

const prompt = `Kamu adalah asisten bisnis profesional. Tulis email pengadaan barang
dalam Bahasa Indonesia yang formal dan sopan kepada supplier berdasarkan data PO berikut:

No. PO   : ${po.noPO}
Supplier : ${po.supplier}
Daftar Barang: ${itemsList}
Total Nilai PO: Rp${total.toLocaleString('id-ID')}

Kembalikan HANYA JSON: { "subject": "...", "body": "..." }`

const completion = await openai.chat.completions.create({

MIS — Dashboard, Analitik, Notifikasi, Laporan

Zentory secara keseluruhan adalah sistem MIS (Management Information System) untuk pengelolaan inventori. Fitur MIS mencakup:

  • Dashboard real-time — KPI utama: total produk, produk di bawah ROP, PO aktif
  • Bar chart — Aktivitas pengadaan 7 hari terakhir
  • Analitik Fast/Slow/Dead Stock — Tabel klasifikasi + rekomendasi tindakan
  • Histori lengkap — PO, Penerimaan, Stock Opname, Sales Return
  • Notifikasi berbasis event — Auto Draft PO, PO Disetujui, Opname Selesai, dll.
